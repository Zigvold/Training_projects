Правила работы с переменными
{
1. Все переменные объвлять перед использованием.
2. Стараться использовать const везде, где это возможно.
3. Внутри фонкций не изменять переменные с внешних областей видимости.
}

Строгий режим - инструкция интерпритатору JavaScript анализировать чей либо код более пристально,
для того , чтобы включить строгий режим достаточно прописать строчку 'use strict' в начале JS файла.
{
Строгий режим запрещает: Использование необъявленных переменных, некорректные синтаксические конструкции,
Строгий режим преобразует "тихие ошибки в исключения"
}

Операторы - встроенные JavaScript функции
{
Арефметические:+,-,*,/
Сравнения:===, !==, <=, >=
Логические: &&, ||, !
Присваевания: =

Текстовые операторы:typeOf, instanceOf, new, delete
Унарные операторы - операторы у которых один аргумент
    {
    a++
    +a
    delete Obj.a
    typeOf a
    new Object()
    }

Бинарные операторы - операторы у которых всегда два аргумента
    {
    a = 5
    a + b
    a += b
    a === b
    a && b
    etc.
    }

Ложные значение в JavaScript - значение которые при привидении к логическому типу возвращают false
Ложные значения:'', -, null, undefined, 0
Приведение к типу Boolean: Boolean(x) -> true/false

Хостинг - это механизм JavaScript, при котором объявления переменных и функций перемещаются в верхнюю часть их области перед выполнением кода.
Неизбежно это означает, что независимо от того, где объявлены функции и переменные, они перемещаются в верхнюю часть их области, независимо от того, является ли их область глобальной или локальной.
Однако следует отметить тот факт, что подъемный механизм только фиксирует объявление. Назначения остаются на месте.
}
